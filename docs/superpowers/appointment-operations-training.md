# Appointment operations — sign-up, slots, and recipes (Phase 2 training)

For appointment/biometric visas only (NOT online eVisa/ETA). Honest note: exact portals, fees, and rules differ
per destination and change — always verify the current official process for each country before you sell it. This
teaches the durable method, not country-specific numbers.

## Part 1 — Sign-up (which accounts, how)

Appointment visas are lodged through an outsourced **visa application centre** or the **embassy** directly. The
three you'll meet:

1. **VFS Global** (vfsglobal.com) — the largest outsourcer; handles many countries' UK visa centres.
2. **TLScontact** (tlscontact.com) — the other big one (used by several Schengen states + others).
3. **Embassy / consulate portal** — some countries take applications directly, not via an outsourcer.

**How to register (the real flow, generic):**
- Go to the **official** centre site for *that specific destination* (each country has its own VFS/TLS landing page — search "VFS [country] UK" / "TLScontact [country] UK", confirm it's the official domain, not a copycat).
- Create an account: email + password; verify the email.
- You'll select the **visa category** + **application centre (city)** — this drives which slots you see.
- Keep one clean operator account per centre/country. Record the login in your password manager + note it in the **Supply-chain registry**.

**Before you can book anything, have ready:** the customer's completed application (reference number from the
government portal where required), passport details, and the document set. Centres often require the online
application to be done *first*, then you book the in-person appointment to lodge it.

> Verify per destination: whether it's VFS / TLS / embassy, the exact category, the fee, and whether biometrics
> are required. Put this in the destination's setup sheet.

## Part 2 — Getting slots (the routine)

Slots are scarce; this is a discipline, not luck.

1. **Pre-stage the account** (Part 1) so you can book the instant a slot shows.
2. **Know the release window.** Centres release new slots in **batches** — commonly **early morning on weekdays, local time**. Log into each centre at its window and check.
3. **Build a daily check routine.** A fixed time each morning to sweep your awaiting-slot orders across their centres.
4. **Be flexible.** Offer the customer the **earliest slot at any reasonable centre/city + date** — flexibility is the single biggest lever.
5. **Book + confirm in minutes.** First acceptable slot → reserve → the order's Appointment box generates the customer pack. Speed matters; slots vanish.
6. **Premium / prime-time / fast-track slots.** Where the centre sells one, **offer it as a paid add-on** — the customer pays the centre's extra fee, you record it (premium-slot box). The sanctioned fast path + pure margin.
7. **Learn the pattern.** Note each centre's release timing/behaviour in the Supply-chain registry so you sharpen.
8. **Set expectations up front.** "Slots vary; we book the earliest legitimately available." Never promise a date.

**Never:** scraping/auto-booking bots. They breach centre terms → account ban → you lose access for **every**
customer. The edge is preparedness + flexibility + premium, full stop.

## Part 3 — A recipe per appointment step (9 lenses)

### Recipe A — Identify centre & find a slot
- **What** find the right centre + an available slot. **Why** it's the bottleneck that sets the timeline. **How** centre account ready → check release windows → flexible date/city. **When** the moment docs are ready. **Where** VFS/TLS/embassy portal. **Which** supply-chain registry + appointment box. **Who** owner. **Would** book the earliest acceptable slot. **Could** premium slot, alternate centre, or courier-submission route where offered.

### Recipe B — Book the appointment
- **What** secure + confirm the slot. **Why** unconfirmed = not booked. **How** reserve, pay any centre fee, capture the appointment reference + date. **When** instantly on finding a slot. **Where** centre portal → order Appointment box (status = booked). **Which** appointments + govt-fields. **Who** owner. **Would** book + record ref/date. **Could** reschedule if the customer can't attend; upgrade to premium.

### Recipe C — Prepare the appointment pack
- **What** tell the customer exactly what to bring + do. **Why** a rejected-on-the-day item wastes the slot. **How** generate the pack (passport, supporting docs, appointment confirmation printout, biometrics note, arrive-early). **When** right after booking. **Where** Appointment box → appointment_booked email. **Which** appointments + emails. **Who** owner. **Would** send the pack + a pre-appointment check call. **Could** add destination-specific items (insurance, photos).

### Recipe D — Customer attends (biometrics)
- **What** the customer lodges in person + gives biometrics. **Why** can't be skipped for these visas. **How** they attend with the pack; fingerprints/photo taken; docs submitted. **When** the booked slot. **Where** the centre. **Which** n/a (customer action). **Who** the customer; owner on standby. **Would** confirm attendance + collect the receipt. **Could** rebook if a document is rejected on the day.

### Recipe E — In-person submission + record
- **What** confirm lodgement + capture references. **Why** the application is now officially in. **How** record the **government reference** + fee-paid; status → submitted. **When** same day as the appointment. **Where** order Govt-submission box. **Which** govt-fields + qa-gate + tracker. **Who** owner. **Would** record everything + send the submitted email. **Could** chase the centre if the receipt/ref is delayed.

### Recipe F — Passport return
- **What** get the passport safely back. **Why** you hold the customer's most important document. **How** **tracked + insured courier (we pay)**; record carrier + tracking; mark dispatched. **When** when the decision/passport is returned by the centre. **Where** order Passport-return box. **Which** passport-return + emails. **Who** ops/owner. **Would** dispatch tracked + notify the customer. **Could** local collection where the customer prefers + it's offered.

## The honest summary
You can run the **entire business** without ever doing this — online visas have no appointments. Add this layer
only when you take on appointment-visa destinations, set up the VFS/TLS account per country, run the daily slot
routine, and offer premium slots. The system already has every box you need; this is the *operating discipline*
around it.
