> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Receive the #4 order payload, create a Pipedrive deal in a visa-fulfilment pipeline, automate stage-driven customer emails, and apply a stage-based refund rule — all backed by a tested refund calculator + deal mapper.

**Architecture:** Two pure functions are tested under Node — the **refund calculator** (deal stage → service-fee % + government-fee rule) and the **payload→deal mapper** (#4 payload → Pipedrive deal fields). The CRM itself (Pipedrive pipeline, fields), the Zapier automations, and the transactional emails are config-and-verify. A small handoff endpoint (or Zapier catch-hook) ingests the payload; documents stay as expiring links.

**Tech Stack:** Pipedrive, Zapier, Postmark or SendGrid (transactional email), Node (pure-logic tests + optional handoff function), the #4 order payload contract.

**Prerequisite:** #4 funnel emits the order payload to `CRM_WEBHOOK_URL`. Pipedrive + Zapier + email-provider accounts. `node -v` ≥18. Paths relative to repo root.

`serverless/crm-core.js`:
```js
// Pure CRM logic: stage-based refund rule + payload->deal mapping. No network.
export const STAGES = {
  PAID: 'paid', AWAITING_DOCS: 'awaiting_docs', DOC_REVIEW: 'doc_review',
  SUBMITTED: 'submitted', AWAITING_DECISION: 'awaiting_decision',
  DELIVERED: 'delivered', WON: 'won', REJECTED: 'rejected', REFUNDED: 'refunded',
};

// Service-fee refund % by stage (govt fee never refundable). Sliding scale from research.
const SERVICE_PCT = {
  paid: 100, awaiting_docs: 100, doc_review: 75,
  submitted: 0, awaiting_decision: 0, delivered: 0, won: 0, rejected: 0,
};

export function refundFor(payload, stage) {
  const pct = SERVICE_PCT[stage];
  if (pct === undefined) throw new Error('unknown stage: ' + stage);
  const serviceLine = (payload.line_items || []).find(li => /service/i.test(li.label));
  const service = serviceLine ? serviceLine.amount_gbp : 0;
  return {
    service_pct: pct,
    service_refund_gbp: Math.round(service * pct) / 100,
    govt_refundable: false,
  };
}

export function payloadToDeal(payload) {
  return {
    title: `${payload.order_ref} — ${payload.dest} ${payload.tier} visa`,
    value: payload.amount_gbp,
    currency: 'GBP',
    stage: STAGES.PAID,
    person: { name: payload.applicant?.name, email: payload.applicant?.email, phone: payload.applicant?.phone },
    custom: {
      order_ref: payload.order_ref,
      product: payload.product,
      dest: payload.dest,
      tier: payload.tier,
      passport_number: payload.passport_number,
      documents: payload.documents || [],
      stripe_session_id: payload.stripe_session_id,
    },
  };
}
```

`serverless/crm-ingest.js`:
```js
import { payloadToDeal } from './crm-core.js';

// Receives the #4 order payload, maps it, creates a Pipedrive deal.
export default async function handler(req, res) {
  if (req.method !== 'POST') return res.status(405).end();
  const deal = payloadToDeal(req.body);
  const token = process.env.PIPEDRIVE_API_TOKEN;
  const base = `https://${process.env.PIPEDRIVE_COMPANY}.pipedrive.com/api/v1`;

  // 1) person
  const personRes = await fetch(`${base}/persons?api_token=${token}`, {
    method: 'POST', headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ name: deal.person.name || deal.person.email, email: deal.person.email, phone: deal.person.phone }),
  }).then(r => r.json());

  // 2) deal (custom field keys must be the Pipedrive hashed keys — map in env CONFIG)
  await fetch(`${base}/deals?api_token=${token}`, {
    method: 'POST', headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      title: deal.title, value: deal.value, currency: deal.currency,
      person_id: personRes.data?.id,
      pipeline_id: Number(process.env.PIPEDRIVE_PIPELINE_ID),
    }),
  });
  return res.status(200).json({ ok: true });
}
```
> Pipedrive custom fields use hashed keys; store the key map in env or a config object and extend the deal body with them. Document the mapping in `pipeline.md`.
